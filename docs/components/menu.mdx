`RemixMenu` displays typed menu content in an overlay. Its trigger is any
widget, and every action, controlled checkbox or radio item, group, label,
separator, and recursively nested submenu implements `RemixMenuItemData<T>`.

## When to use this

- **Action menus**: Show actions for the current item or selection
- **Context menus**: Present commands that depend on local context
- **Settings**: Mix actions with controlled checkbox and radio preferences
- **Nested navigation**: Organize related commands into submenus

## Basic implementation

<CodeGroup title="Compositional menu" defaultLanguage="dart">
```dart
import 'package:flutter/material.dart';
import 'package:remix/remix.dart';

class MenuExample extends StatefulWidget {
  const MenuExample({super.key});

  @override
  State<MenuExample> createState() => _MenuExampleState();
}

class _MenuExampleState extends State<MenuExample> {
  final _controller = MenuController();
  bool _showGrid = true;
  String _density = 'comfortable';

  @override
  Widget build(BuildContext context) {
    return RemixMenu<String>(
      controller: _controller,
      trigger: Container(
        constraints: const BoxConstraints(minHeight: 40),
        padding: const EdgeInsets.symmetric(horizontal: 14),
        alignment: Alignment.center,
        child: const Text('Open menu'),
      ),
      items: [
        const RemixMenuLabel(child: Text('View')),
        const RemixMenuAction(
          value: 'history',
          leading: Icon(Icons.history),
          child: Text('History'),
        ),
        RemixMenuCheckboxItem(
          value: 'show-grid',
          checked: _showGrid,
          onChanged: (checked) => setState(() => _showGrid = checked),
          child: const Text('Show grid'),
        ),
        const RemixMenuSeparator(),
        RemixMenuRadioGroup<String>(
          value: _density,
          onChanged: (value) => setState(() => _density = value),
          children: const [
            RemixMenuRadioItem(
              value: 'compact',
              child: Text('Compact'),
            ),
            RemixMenuRadioItem(
              value: 'comfortable',
              child: Text('Comfortable'),
            ),
          ],
        ),
        const RemixMenuSubmenu<String>(
          child: Text('More'),
          items: [
            RemixMenuAction(value: 'settings', child: Text('Settings')),
          ],
        ),
      ],
      positioning: const OverlayPositionConfig(
        side: OverlaySide.bottom,
        alignment: OverlayAlignment.start,
        sideOffset: 8,
      ),
      onSelected: (value) => debugPrint('Selected: $value'),
      style: RemixMenuStyler()
          .overlay(
            .padding(EdgeInsetsGeometryMix.all(8))
                .color(Colors.white)
                .borderRadiusAll(const Radius.circular(12)),
          )
          .item(
            .spacing(8)
                .borderRadiusAll(const Radius.circular(8))
                .onHovered(
                  .color(Colors.blueGrey.shade50),
                ),
          ),
    );
  }
}
```
</CodeGroup>

## Fortal widgets

`FortalMenu` applies the Radix-aligned content and item recipes while retaining
the same compositional item API and arbitrary trigger.

<CodeGroup title="Fortal variants" defaultLanguage="dart">
```dart
import 'package:flutter/material.dart';
import 'package:remix/remix.dart';

class FortalMenuExample extends StatelessWidget {
  const FortalMenuExample({super.key});

  static const _items = <RemixMenuItemData<String>>[
    RemixMenuAction(
      value: 'edit',
      leading: Icon(Icons.edit),
      child: Text('Edit'),
    ),
    RemixMenuAction(
      value: 'delete',
      leading: Icon(Icons.delete),
      child: Text('Delete'),
    ),
  ];

  @override
  Widget build(BuildContext context) {
    return Row(
      spacing: 16,
      children: [
        FortalMenu<String>.solid(
          trigger: const Text('Solid'),
          items: _items,
          onSelected: debugPrint,
        ),
        FortalMenu<String>.soft(
          trigger: const Text('Soft'),
          items: _items,
          onSelected: debugPrint,
        ),
      ],
    );
  }
}
```

## Item types

Every item type below implements `RemixMenuItemData<T>`, preserving the
original typed collection while supporting compositional content.

- `RemixMenuAction<T>` activates a typed value and accepts arbitrary `child`,
  `leading`, and `trailing` widgets.
- `RemixMenuCheckboxItem<T>` is controlled with `checked` and `onChanged`.
- `RemixMenuRadioGroup<T>` controls typed `RemixMenuRadioItem<T>` children.
- `RemixMenuGroup`, `RemixMenuLabel`, and `RemixMenuSeparator` add structure and
  accessibility boundaries.
- `RemixMenuSubmenu<T>` accepts its own compositional `items` and may be
  nested recursively.

Items close on activation by default. Set an item's `closeOnActivate` to false
when interaction should keep the menu open. Disabled items remain discoverable
but cannot activate.

## Styling

`RemixMenuStyler` owns `overlay`, `containerEffects`, default `item`, `label`,
and `divider` slots. Each item's `style` merges after the menu defaults. The
`trigger` slot applies to `RemixMenuTrigger`; style arbitrary trigger widgets
independently.
