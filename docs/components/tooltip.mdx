`RemixTooltip` presents contextual information on hover, focus, or touch. It
supports controlled open state, collision-aware positioning, hoverable content,
root overlays, optional arrows, and configurable transitions.

## Basic implementation

<CodeGroup title="Tooltip with arrow" defaultLanguage="dart">
```dart
import 'package:flutter/material.dart';
import 'package:remix/remix.dart';

class TooltipExample extends StatelessWidget {
  const TooltipExample({super.key});

  @override
  Widget build(BuildContext context) {
    return RemixTooltip(
      tooltipChild: const Text('Archive this message'),
      tooltipSemantics: 'Archive this message',
      positioning: const OverlayPositionConfig(
        side: OverlaySide.top,
        alignment: OverlayAlignment.center,
        sideOffset: 6,
      ),
      showArrow: true,
      maxWidth: 240,
      style: RemixTooltipStyler()
          .paddingX(12)
          .paddingY(8)
          .backgroundColor(Colors.black87)
          .arrowColor(Colors.black87)
          .label(.color(Colors.white)),
      child: const Icon(Icons.archive_outlined),
    );
  }
}
```
</CodeGroup>

## Controlled state

Pass `open` and `onOpenChanged` to control visibility. Leave `open` null for
internal state. User requests still flow through `onOpenChanged`, which makes it
possible to coordinate multiple tooltips without replacing trigger behavior.

## Fortal widget

<CodeGroup title="Fortal tooltip" defaultLanguage="dart">

const helpfulTooltip = FortalTooltip(

</CodeGroup>

Radix color roles, and a maximum width of 360 logical pixels.

## Constructor

<CodeGroup title="RemixTooltip constructor" defaultLanguage="dart">
```dart
const RemixTooltip({
  Key? key,
  required Widget tooltipChild,
  required Widget child,
  String? tooltipSemantics,
  OverlayPositionConfig positioning = const OverlayPositionConfig(),
  bool? open,
  ValueChanged<bool>? onOpenChanged,
  bool disableHoverableContent = false,
  bool enableTapToDismiss = true,
  TooltipTriggerMode triggerMode = TooltipTriggerMode.longPress,
  bool enableFeedback = true,
  TooltipTriggeredCallback? onTriggered,
  AnimationStyle animationStyle = const AnimationStyle(
    curve: Curves.fastOutSlowIn,
    duration: Duration(milliseconds: 150),
    reverseDuration: Duration(milliseconds: 75),
  ),
  bool useRootOverlay = false,
  bool excludeSemantics = false,
  double? minWidth,
  double? maxWidth,
  bool showArrow = false,
  Size arrowSize = const Size(10, 5),
  RemixTooltipStyler style = const RemixTooltipStyler.create(),
  RemixTooltipSpec? styleSpec,
})
```
</CodeGroup>

## Timing and interaction

- `waitDuration` is the mouse-hover delay.
- `showDuration` controls how long a touch-triggered tooltip remains open after
  activation.
- `dismissDuration` is the hover-exit grace period.
- `disableHoverableContent: false` lets a pointer move into tooltip content
  without closing it. `enableTapToDismiss` controls outside-tap dismissal.
- `useRootOverlay` chooses the root rather than nearest overlay.
- `showArrow` paints toward the resolved side when `arrowColor` is styled. The
  arrow follows the trigger after collision flipping or cross-axis shifting,
  including correctly rotated left/right geometry.

## Semantics

`tooltipSemantics` supplies the trigger's accessible description. Set
`excludeSemantics` only when equivalent information is already provided by the
surrounding control. The overlay does not duplicate trigger semantics.
