`RemixSlider` is a controlled, arbitrary multi-thumb slider. Values remain in
ascending order, thumbs cannot cross, and each thumb has independent focus and
accessibility semantics.

## When to use this

- **Single values**: Volume, brightness, or progress controls
- **Ranges**: Minimum and maximum prices or dates
- **Multiple thresholds**: Ordered stops within one numeric domain
- **Vertical controls**: Use `orientation` and `inverted` for alternate layouts

## Basic implementation

<CodeGroup title="Two-thumb range" defaultLanguage="dart">
```dart
import 'package:flutter/material.dart';
import 'package:remix/remix.dart';

class SliderExample extends StatefulWidget {
  const SliderExample({super.key});

  @override
  State<SliderExample> createState() => _SliderExampleState();
}

class _SliderExampleState extends State<SliderExample> {
  List<double> _values = [20, 80];

  @override
  Widget build(BuildContext context) {
    return SizedBox(
      width: 300,
      child: RemixSlider(
        values: _values,
        min: 0,
        max: 100,
        step: 5,
        minSpacing: 10,
        semanticLabels: const ['Minimum price', 'Maximum price'],
        onChanged: (values) => setState(() => _values = values),
        style: RemixSliderStyler()
            .thumbSize(const Size(24, 24))
            .thumb(.shape(CircleBorderMix()))
            .thumbColor(Colors.black)
            .trackColor(Colors.grey.shade300)
            .rangeColor(Colors.black)
            .thickness(2),
      ),
    );
  }
}
```
</CodeGroup>

For one thumb, pass a one-element list and read `values.single` in the callback.

<CodeGroup title="Fortal variants" defaultLanguage="dart">
```dart
import 'package:flutter/material.dart';
import 'package:remix/remix.dart';

class FortalSliderExample extends StatefulWidget {
  const FortalSliderExample({super.key});

  @override
  State<FortalSliderExample> createState() => _FortalSliderExampleState();
}

class _FortalSliderExampleState extends State<FortalSliderExample> {
  List<double> _values = [35];

  @override
  Widget build(BuildContext context) {
    return FortalSlider(
      variant: FortalSliderVariant.surface,
      values: _values,
      semanticLabels: const ['Volume'],
      onChanged: (values) => setState(() => _values = values),
    );
  }
}
```

## Controlled-value contract

- `min` must be less than `max`; `step` must be positive; `minSpacing` must be
  non-negative.
- `values` must be finite, in range, aligned to `step`, ascending, and separated
  by at least `minSpacing`.
- Callbacks receive the complete normalized list. Omitting `onChanged` preserves
  rendering while preventing value changes.
- `focusNodes`, `semanticLabels`, and `semanticFormatterCallbacks`, when
  supplied, correspond by index to the value list.
- Pointer input chooses the nearest eligible thumb. Keyboard input acts on the
  focused thumb and respects orientation, inversion, steps, and neighbors.

## Styling

`RemixSliderStyler` exposes container, track, range, and thumb styles plus
surface/overlay paint layers. Convenience methods such as `trackColor`,
`rangeColor`, `thumbColor`, `thumbSize`, and `thickness` configure the common
case while state variants handle hover, drag, focus, and disabled styling.
