`RemixTabs` coordinates a tab list and its panels. Tab buttons are actual
children of `RemixTabBar`, so the list can wrap, justify, and change orientation
without requiring callers to construct a Row or Column.

## When to use this

- **Content organization**: Switch between peer sections in one context
- **Settings panels**: Group related controls without navigating away
- **Responsive tab lists**: Wrap or justify tabs when space is constrained
- **Keyboard navigation**: Provide roving focus and manual or automatic activation

## Basic implementation

<CodeGroup title="Controlled tabs" defaultLanguage="dart">
```dart
import 'package:flutter/material.dart';
import 'package:remix/remix.dart';

class TabsExample extends StatefulWidget {
  const TabsExample({super.key});

  @override
  State<TabsExample> createState() => _TabsExampleState();
}

class _TabsExampleState extends State<TabsExample> {
  String _tab = 'account';

  @override
  Widget build(BuildContext context) {
    return RemixTabs(
      selectedTabId: _tab,
      onChanged: (id) => setState(() => _tab = id),
      child: Column(
        crossAxisAlignment: CrossAxisAlignment.stretch,
        children: [
          RemixTabBar(
            wrap: RemixTabBarWrap.wrap,
            justify: RemixTabBarJustify.start,
            children: const [
              RemixTab(tabId: 'account', label: 'Account'),
              RemixTab(tabId: 'security', label: 'Security'),
            ],
          ),
          const RemixTabView(
            tabId: 'account',
            child: Text('Account settings'),
          ),
          const RemixTabView(
            tabId: 'security',
            child: Text('Security settings'),
          ),
        ],
      ),
    );
  }
}
```
</CodeGroup>

## Fortal widgets

<CodeGroup title="Fortal tabs" defaultLanguage="dart">
```dart
Widget fortalTabs({
  required String selectedTab,
  required ValueChanged<String> onChanged,
}) => RemixTabs(
  selectedTabId: selectedTab,
  onChanged: onChanged,
  child: Column(
    children: [
      const FortalTabBar(
        size: FortalTabsSize.size2,
        children: [
          FortalTab(tabId: 'account', label: 'Account'),
          FortalTab(tabId: 'security', label: 'Security'),
        ],
      ),
      const FortalTabView(
        tabId: 'account',
        child: Text('Account settings'),
      ),
      const FortalTabView(
        tabId: 'security',
        child: Text('Security settings'),
      ),
    ],
  ),
);
```
</CodeGroup>

`FortalTabBar` provides the size, color, and high-contrast scope consumed by its
`FortalTab` descendants. `FortalTabView` supplies the matching panel behavior.

## Layout and behavior

- `RemixTabs` requires either a controller or `selectedTabId`. Its `orientation`
  controls list layout, arrow-key direction, and tab-list semantics.
- With `activationMode: .automatic`, moving focus also selects the tab. In
  `.manual` mode, arrow keys move focus without changing the panel until Space
  or Enter explicitly activates the focused tab.
- `RemixTabBarWrap.nowrap` keeps one line; `wrap` and `wrapReverse` create runs.
  `justify` aligns tabs along the main axis or each run.
- Tabs use roving focus, and `onEscapePressed` provides an explicit escape path.
- `RemixTabView` exposes only the selected panel. With `maintainState: true`, an
  inactive panel keeps element state while remaining hidden and nonsemantic.

## Styling

`RemixTabBarStyler`, `RemixTabStyler`, and `RemixTabViewStyler` own the list,
tab, and panel surfaces respectively. Tab state variants include hovered,
pressed, focused, selected, and disabled.
