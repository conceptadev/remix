`RemixSelect` is a controlled single-value input. Its sealed `entries` model
supports selectable items, groups, labels, and separators without allowing
arbitrary interactive widgets inside listbox content.

## When to use this

- **Form inputs**: Collect one value from a known set
- **Filters**: Choose one category, status, or sort order
- **Settings**: Select one configuration value
- **Grouped choices**: Organize longer lists with labels and separators

## Basic implementation

<CodeGroup title="Structured select" defaultLanguage="dart">
```dart
import 'package:flutter/material.dart';
import 'package:remix/remix.dart';

class SelectExample extends StatefulWidget {
  const SelectExample({super.key});

  @override
  State<SelectExample> createState() => _SelectExampleState();
}

class _SelectExampleState extends State<SelectExample> {
  String? _selectedValue;

  @override
  Widget build(BuildContext context) {
    return SizedBox(
      width: 240,
      child: RemixSelect<String>(
        trigger: const RemixSelectTrigger(placeholder: 'Choose a fruit'),
        entries: const [
          RemixSelectLabel(label: 'Fresh'),
          RemixSelectGroup(
            semanticLabel: 'Fresh fruit',
            entries: [
              RemixSelectItem(value: 'apple', label: 'Apple'),
              RemixSelectItem(value: 'banana', label: 'Banana'),
            ],
          ),
          RemixSelectSeparator(),
          RemixSelectItem(
            value: 'orange',
            label: 'Orange',
            enabled: false,
          ),
        ],
        selectedValue: _selectedValue,
        onChanged: (value) => setState(() => _selectedValue = value),
        style: RemixSelectStyler()
            .trigger(
              RemixSelectTriggerStyler()
                  .paddingY(10)
                  .paddingX(12)
                  .borderAll(color: const Color(0xFF898988))
                  .borderRadiusAll(const Radius.circular(12)),
            )
            .content(
              RemixSelectContentStyler()
                  .width(240)
                  .paddingAll(6)
                  .color(Colors.white)
                  .borderRadiusAll(const Radius.circular(12)),
            ),
      ),
    );
  }
}
```
</CodeGroup>

## Fortal widgets

`FortalSelect` styles the trigger and popup independently. Use
`triggerVariant` for `classic`, `surface`, `soft`, or `ghost`, and
`contentVariant` for `solid` or `soft` content.

<CodeGroup title="Fortal variants" defaultLanguage="dart">
```dart
import 'package:flutter/material.dart';
import 'package:remix/remix.dart';

class FortalSelectExample extends StatefulWidget {
  const FortalSelectExample({super.key});

  @override
  State<FortalSelectExample> createState() => _FortalSelectExampleState();
}

class _FortalSelectExampleState extends State<FortalSelectExample> {
  String? _value;

  static const _entries = <RemixSelectEntry<String>>[
    RemixSelectItem(value: 'opt1', label: 'Option 1'),
    RemixSelectItem(value: 'opt2', label: 'Option 2'),
  ];

  @override
  Widget build(BuildContext context) {
    return FortalSelect<String>(
      triggerVariant: FortalSelectTriggerVariant.surface,
      contentVariant: FortalSelectContentVariant.solid,
      trigger: const RemixSelectTrigger(placeholder: 'Choose an option'),
      entries: _entries,
      selectedValue: _value,
      onChanged: (value) => setState(() => _value = value),
    );
  }
}
```
</CodeGroup>

## Entry types

- `RemixSelectItem<T>` is a selectable typed value with a visible label.
- `RemixSelectGroup<T>` creates a semantic group around nested entries.
- `RemixSelectLabel<T>` is a non-selectable heading.
- `RemixSelectSeparator<T>` is a decorative boundary.

The selected value is controlled by `selectedValue`. A null `onChanged` ignores
selection requests, but an enabled select may still open so users can inspect
its options. Set `enabled: false` to disable opening and selection together.
The overlay is uncontrolled when `open` is null. With a non-null `open`, user
interaction only reports the requested state through `onOpenChanged`; the
owner must rebuild with the new value, so rejecting a request leaves the
overlay unchanged. `FortalSelect` forwards the same controlled-open contract.

## Styling

`RemixSelectStyler` owns separate `trigger`, `content`, `item`, `label`, and
`separator` slots. Per-item and per-label styles merge after those defaults.
The wrapper and glyph builders replace only presentation; Remix retains the
select behavior and semantics.
